## Reglas

1. **No tocar código.** Sólo archivos de documentación (.md, .txt, .rst, etc.).
2. **No crear docs gigantes.** Resumen ejecutivo antes que enciclopedia.
3. **No duplicar documentos.** Si la información ya existe, referenciarla, no copiarla.
4. **No contradecir fuentes vigentes.** Si hay contradicción, reportar, no inventar.
5. **Breve y preciso.** Cada documento debe tener un propósito claro.

## Prohibido

- Editar archivos de código (.py, .js, .ts, etc.)
- Crear documentos que repitan información existente
- Escribir más de lo necesario para el objetivo
- Modificar README principal sin autorización
- `git add .`
- `git push` sin autorización
- Generar "documentación por documentar"

## Validación mínima

- No se tocaron archivos de código
- La información es consistente con fuentes existentes
- El documento cumple el objetivo del episodio
- No hay duplicación con documentos existentes

## Criterios de rollback

- Se tocaron archivos de código
- El documento contradice fuentes vigentes
- El documento es excesivamente largo sin justificación
- Se duplicó información existente

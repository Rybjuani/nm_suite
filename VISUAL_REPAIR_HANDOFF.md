## How to read this checklist

Each line is one surface_key with its current state. The state can be:

- `[ ]` — OPEN, not yet closed under V2
- `[x]` — CLOSED under V2 (with evidence record path + commit hash)
- `[~]` — BLOCKED (in `closure_policy.yaml:known_non_deterministic_surfaces` or
  `state_ambiguous_pairs`; requires special handling)

The previous V1 closure metadata (severity, findings, changed_pixel_ratio,
evidence hashes from the CRLF era) is preserved in
`docs/_archive/V1_CLOSURE_HISTORY.md` for forensic reference. It is NOT
authoritative.
